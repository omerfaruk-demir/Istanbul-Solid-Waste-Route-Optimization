# EDA output files — what each file is for

All files in this folder are **generated by** [`notebooks/eda.ipynb`](../../notebooks/eda.ipynb). **Original data under `datas/` is never modified.**

---

## CSV files

### `nodes.csv`

Master node table for the capacitated vehicle routing (CVRP) network.

| Column | Meaning |
|--------|---------|
| `node_id` | Identifier: `D0` (depot), `S1`…`S9` (transfer stations), `F1`…`F9` (disposal/processing sites). |
| `name` | Facility name from source data. |
| `lat`, `lon` | WGS84 coordinates. |
| `node_type` | `Depot`, `Transfer Station`, or `Disposal`. |
| `demand_tonnes` | Monthly waste attributed to that station (tonnes). Depots and disposal nodes have `0`. Stations hold values aggregated from municipality-level volumes via nearest-centroid assignment. |
| `side` | `Avrupa` or `Anadolu` (used for fleet-side constraints). |

---

### `distance_matrix_km.csv`

Square matrix (rows and columns = `node_id` order matching `nodes.csv`). Cell `[i][j]` is **great-circle (Haversine) distance in kilometres** between node *i* and node *j*. Symmetric; diagonal is zero. Used as a baseline travel-distance cost between any pair of nodes.

---

### `time_matrix_baseline_min.csv`

Same indexing as `distance_matrix_km.csv`. Cell `[i][j]` is **estimated travel time in minutes** under **free-flow**, computed from distance and a baseline speed `v_free` (see `alpha_r.json`). Formula conceptually: `(distance_km / v_free_kmh) × 60`. Used for time-window style reasoning and comparison with congestion-adjusted times.

---

### `fleet.csv`

Yearly garbage-truck counts copied from `cop-kamyonu-says.xlsx` in a normalised tabular form.

| Column | Meaning |
|--------|---------|
| `year` | Calendar year. |
| `asia` | Trucks on the Anadolu side. |
| `europe` | Trucks on the Avrupa side. |
| `total` | Sum of the two sides. |

The **2025** row is the natural reference for maximum fleet size in the model (proposal uses current-year counts).

---

## JSON files

### `alpha_r.json`

Hourly **traffic congestion multipliers** aligned with the proposal notation **α_r** (ratio-style scaling of travel time versus free-flow).

| Field | Meaning |
|-------|---------|
| `v_free_kmh` | Free-flow reference speed (km/h), derived from traffic data (e.g. high quantile of speeds in low-traffic night hours). |
| `alpha_r` | Object keyed by hour `"0"`…`"23"`. Each value is **α for that hour**: effective speeds are lower when **α** is larger (more congestion). |

**Interpretation:** at hour *h*, a simple scaling view is  
`effective_speed_kmh ≈ v_free_kmh / alpha_r[h]`  
so travel time grows roughly proportional to **α** when distance is fixed.

---

### `cvrp_instance.json`

Single bundle meant as **input to routing / optimisation code** (e.g. `optimization.ipynb`) without re-reading raw spreadsheets.

| Key | Contents |
|-----|-----------|
| `depot` | Node `D0`: id, name, coordinates, type. |
| `transfer_stations` | List of `S*` nodes: coordinates, **monthly** `total_waste_tonnes`, `total_trips`, `side`. |
| `disposal_facilities` | List of `F*` nodes: coordinates and `side`. |
| `fleet_2025` | `asia`, `europe`, `total` truck counts for 2025. |
| `alpha_r` | Same hourly factors as in `alpha_r.json`. |
| `v_free_kmh` | Same baseline speed as in `alpha_r.json`. |

## Limitations (for reporting)

- **Distances/times** use Haversine + scalar speed, not a road network (OSRM/Google-style matrices would be more realistic).
- **Demand**: municipality totals are assigned to the nearest physical station using approximate district centroids; boundary errors are possible.
- **Traffic**: January 2025 only; **α_r** may not generalise to other months without recomputation.
